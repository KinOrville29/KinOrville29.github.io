本文将详细介绍如何使用 Hugo 静态网站生成器配合 Blowfish 主题创建个人博客，并将其部署到 GitHub Pages 实现免费访问。

环境准备
#

在开始之前，需要在本地计算机上完成必要的环境配置。

安装 Git
#

Git 是版本控制工具，也是后续部署的必要条件。

访问 Git 官网 (git-scm.com) 下载并安装对应操作系统的版本。安装完成后，在终端中执行以下命令验证安装是否成功：

git --version

安装 Hugo
#

Hugo 是一个用 Go 语言编写的静态网站生成器，以其生成速度快著称。

访问 Hugo GitHub 发布页面 (github.com/gohugoio/hugo/releases) 下载适合您操作系统的版本。或者，如果您使用 macOS 和 Homebrew，可以通过以下命令安装：

brew install hugo

安装完成后，验证 Hugo 是否安装成功：

hugo version

确保安装的是 Hugo Extended 版本，这对 Blowfish 主题的 Sass/SCSS 处理是必需的。

创建 Hugo 网站
#

完成环境准备后，可以开始创建 Hugo 网站。

初始化新网站
#

在终端中，导航到您希望创建网站的目录，然后执行：

hugo new site myblog
cd myblog

这将创建一个名为 “myblog” 的新目录，其中包含 Hugo 网站的基本结构。

初始化 Git 仓库
#

进入网站目录后，将其初始化为 Git 仓库：

git init

安装 Blowfish 主题
#

Blowfish 是一个设计优雅、功能丰富的 Hugo 主题。

添加主题为子模块
#

推荐将主题添加为 Git 子模块，这样可以方便地更新：

git submodule add https://github.com/nunocoracao/blowfish.git themes/blowfish

基础配置
#

在网站根目录下创建 hugo.toml 配置文件（Hugo v0.110.0 及以上版本使用此名称，旧版本使用 config.toml），并添加以下基本配置：

title = "我的个人博客"
theme = "blowfish"

[params]
description = "这是我的个人博客网站"

配置 Blowfish 主题
#

Blowfish 主题提供了丰富的自定义选项。

基本参数配置
#

在 hugo.toml 中添加更多主题配置：

[params]
description = "分享技术思考和生活感悟"
copyright = "© 2023 我的博客. 保留所有权利."
author = "您的名字"

# 主题特色配置
[params.article]
  readingTime = true  # 显示阅读时间
  toc = true  # 显示目录

# 主页配置
[params.home]
  layout = "profile"  # 主页布局类型

菜单配置
#

创建网站菜单。首先在 config/_default 目录下创建 menus.toml 文件：

[[main]]
  name = "首页"
  pageRef = "/"
  weight = 1

[[main]]
  name = "文章"
  pageRef = "/posts"
  weight = 2

[[main]]
  name = "关于"
  pageRef = "/about"
  weight = 3

然后创建对应的内容目录：

hugo new about.md

编辑 content/about.md，将 draft: true 改为 draft: false。

创建内容
#

现在可以开始为博客添加内容了。

创建第一篇文章
#

使用 Hugo 命令创建新文章：

hugo new posts/我的第一篇文章.md

这将在 content/posts 目录下创建一个新的 Markdown 文件。打开该文件，您会看到类似以下的内容：

---
title: "我的第一篇文章"
date: 2023-04-01T15:30:00+08:00
draft: true
---

这里是文章内容...

将 draft: true 改为 draft: false，然后在 --- 下方添加您的文章内容。

文章前置参数
#

Blowfish 主题支持多种文章前置参数，可以增强文章显示效果：

---
title: "我的第一篇文章"
date: 2023-04-01T15:30:00+08:00
draft: false
description: "这篇文章介绍了如何搭建 Hugo 博客"
tags: ["Hugo", "Blowfish", "GitHub Pages"]
categories: ["技术"]
cover:
  image: "images/cover.jpg"  # 封面图片
  alt: "文章封面"
---

本地测试
#

在部署之前，先在本地测试网站效果。

启动开发服务器
#

在网站根目录下执行：

hugo server -D

-D 参数表示包含草稿文章。执行后，终端会显示服务器地址，通常是 http://localhost:1313。

在浏览器中打开该地址，即可查看网站效果。开发服务器支持热重载，当您修改内容时，浏览器会自动刷新。

构建静态文件
#

当您准备发布网站时，使用以下命令生成静态文件：

hugo

这会在 public 目录下生成所有静态文件，这些文件可以直接部署到任何 Web 服务器。

部署到 GitHub Pages
#

GitHub Pages 提供免费的静态网站托管服务，非常适合 Hugo 网站。

创建 GitHub 仓库
#

在 GitHub 上创建一个新仓库，命名为 您的用户名.github.io。例如，如果您的 GitHub 用户名是 “john”，则仓库名应为 “john.github.io”。

创建部署脚本
#

在网站根目录下创建 .github/workflows/gh-pages.yml 文件：

name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  build-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v3
        with:
          submodules: true

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: 'latest'
          extended: true

      - name: Build
        run: hugo --minify

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          personal_token: ${{ secrets.GITHUB_TOKEN }}
          external_repository: 您的用户名/您的用户名.github.io
          publish_branch: main
          publish_dir: ./public
          force_orphan: true

注意将 您的用户名 替换为实际的 GitHub 用户名。

配置仓库
#

在源代码仓库的 Settings -> Pages 中，将 Source 设置为 “GitHub Actions”。

推送代码
#

将本地代码推送到 GitHub 仓库：

git add .
git commit -m "初始提交"
git branch -M main
git remote add origin https://github.com/您的用户名/您的源代码仓库名.git
git push -u origin main

GitHub Actions 会自动构建并部署您的网站。完成后，可以通过 https://您的用户名.github.io 访问您的博客。

自定义与优化
#

修改主题颜色
#

Blowfish 主题支持多种颜色方案。在 hugo.toml 中添加：

[params]
  colorScheme = "blowfish"  # 可选: blowfish, avocado, etc.

添加评论系统
#

要添加评论功能，可以使用 Giscus（基于 GitHub Discussions）：

[params.giscus]
  repo = "您的用户名/您的仓库名"
  repoID = "您的仓库ID"
  category = "Announcements"
  categoryID = "您的分类ID"

配置搜索功能
#

Blowfish 主题内置了搜索功能，需要额外配置：

[services.algolia]
  index = "您的索引名"
  appID = "您的应用ID"
  searchKey = "您的搜索密钥"

常见问题
#

页面显示 404 错误
#

确保在 GitHub Pages 设置中正确配置了发布源，并且 GitHub Actions 工作流已成功运行。

样式丢失
#

检查是否使用了 Hugo Extended 版本，这是处理 Blowfish 主题 SCSS 文件所必需的。

菜单不显示
#

确认 menus.toml 文件位于 config/_default 目录下，并且页面引用正确。

总结
#

通过本教程，您已经成功使用 Hugo 和 Blowfish 主题创建了一个个人博客，并将其部署到 GitHub Pages。您现在可以：

  1. 在本地编写 Markdown 格式的文章
  2. 使用 Git 管理网站版本
  3. 通过 GitHub Actions 自动部署更新
  4. 通过自定义配置调整网站外观和功能

Hugo 配合 GitHub Pages 提供了一个强大且免费的博客解决方案，适合技术人员和个人博主使用。随着需求的增加，您可以进一步探索 Hugo 和 Blowfish 主题的更多高级功能，如图片处理、短代码和多语言支持等。
